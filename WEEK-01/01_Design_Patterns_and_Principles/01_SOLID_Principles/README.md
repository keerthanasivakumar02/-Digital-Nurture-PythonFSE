# Design Patterns and Principles

## Introduction

This module focuses on the core principles and design patterns used in object-oriented software development. Understanding these concepts helps developers create applications that are easier to maintain, extend, test, and reuse. The module includes the SOLID principles and several commonly used design patterns grouped into different categories.

---

## Contents

### SOLID Principles

- Single Responsibility Principle (SRP)
- Open/Closed Principle (OCP)
- Liskov Substitution Principle (LSP)
- Interface Segregation Principle (ISP)
- Dependency Inversion Principle (DIP)

### Creational Design Patterns

- Singleton Pattern
- Factory Method Pattern
- Builder Pattern

### Structural Design Patterns

- Adapter Pattern
- Decorator Pattern
- Proxy Pattern

### Behavioral Design Patterns

- Observer Pattern
- Strategy Pattern
- Command Pattern

### Architectural Design Patterns

- Model-View-Controller (MVC)
- Dependency Injection (DI)

---

## Objectives

After completing this module, you will be able to:

- Understand the purpose of the SOLID design principles.
- Build applications using commonly used design patterns.
- Write modular and well-structured Python code.
- Improve software quality through reusable and maintainable designs.
- Select suitable design patterns based on application requirements.

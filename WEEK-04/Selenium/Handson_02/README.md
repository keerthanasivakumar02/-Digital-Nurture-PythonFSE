# Hands-On 2 - SDLC vs TDLC, V-Model & Agile QA Integration

## Objective

This hands-on focuses on understanding the relationship between the Software Development Life Cycle (SDLC) and the Testing Life Cycle (TDLC). It also introduces the V-Model, Agile QA practices, Shift-Left Testing, and writing Acceptance Criteria using Gherkin.

## Task 1 - V-Model Mapping

### Topics Covered

- Software Development Life Cycle (SDLC)
- Testing Life Cycle (TDLC)
- V-Model
- SDLC ↔ TDLC Mapping
- Test Artifacts
- Entry Criteria
- Exit Criteria
- Early QA Engagement

### Learning Outcome

- Understand how each development phase maps to a testing phase.
- Learn the importance of planning testing early.
- Understand entry and exit criteria for different testing levels.
- Identify where QA contributes before coding begins.

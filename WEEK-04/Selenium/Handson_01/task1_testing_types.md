# Task 1 - Software Testing Types

## What is Software Testing?

Software testing is the process of verifying and validating that a software application works as expected and satisfies user requirements.

---

## Types of Software Testing

### 1. Unit Testing

**Definition:**
Unit Testing verifies an individual function, method, or module independently.

**Performed By:**
Developer

**Example:**
Testing whether a calculator's `add()` function correctly returns the sum of two numbers.

---

### 2. Integration Testing

**Definition:**
Integration Testing verifies that multiple modules work together correctly.

**Performed By:**
Developer / QA Engineer

**Example:**
Verifying that the Login page correctly communicates with the database to authenticate users.

---

### 3. System Testing

**Definition:**
System Testing verifies the complete application as a whole.

**Performed By:**
QA Engineer

**Example:**
Testing the entire online shopping application, including login, product search, cart, payment, and logout.

---

### 4. User Acceptance Testing (UAT)

**Definition:**
User Acceptance Testing ensures the application satisfies business requirements before release.

**Performed By:**
Client / End User

**Example:**
A customer verifies that an e-commerce website meets all business requirements before it goes live.

---

## Functional Testing

Functional Testing verifies **what the application does**.

Examples:
- Login
- Registration
- Search
- Payment
- Logout

---

## Non-Functional Testing

Non-Functional Testing verifies **how well the application performs**.

Examples:
- Performance Testing
- Load Testing
- Stress Testing
- Security Testing
- Usability Testing

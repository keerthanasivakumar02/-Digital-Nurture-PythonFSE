# Task 4 - Defect Life Cycle

## What is a Defect?

A defect (bug) is a flaw or error in a software application that causes it to behave differently from the expected result.

Example:
Clicking the Login button with valid credentials displays an error instead of opening the Dashboard.

---

## Defect Life Cycle

The Defect Life Cycle describes the stages a defect goes through from the time it is identified until it is closed.

### Stages

1. New
2. Assigned
3. Open
4. Fixed
5. Retest
6. Closed

If the issue still exists after retesting:

7. Reopened

---

## Stage Explanation

### 1. New

The tester identifies a defect and reports it.

Example:
The Login button is not working.

---

### 2. Assigned

The Test Lead or Project Manager assigns the defect to a developer.

---

### 3. Open

The developer starts analyzing and fixing the defect.

---

### 4. Fixed

The developer fixes the issue and updates the defect status to Fixed.

---

### 5. Retest

The QA Engineer retests the application using the same test case.

---

### 6. Closed

If the issue is resolved successfully, the defect is marked as Closed.

---

### 7. Reopened

If the issue still exists, the tester changes the status to Reopened and assigns it back to the developer.

---

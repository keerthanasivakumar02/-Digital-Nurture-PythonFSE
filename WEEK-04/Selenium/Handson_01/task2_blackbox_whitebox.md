# Task 2 - Black Box Testing vs White Box Testing

## What is Black Box Testing?

Black Box Testing is a software testing technique where the tester checks the functionality of the application without knowing its internal code or implementation.

### Characteristics

- No knowledge of source code is required.
- Focuses on inputs and outputs.
- Tests application functionality.
- Performed by QA Engineers.

### Example

A tester checks whether the Login page accepts a valid username and password and redirects the user to the dashboard.

The tester does not know how the login code is written.

---

## What is White Box Testing?

White Box Testing is a software testing technique where the tester has knowledge of the application's internal code, logic, and structure.

### Characteristics

- Requires programming knowledge.
- Tests code logic, loops, and conditions.
- Performed mainly by Developers.

### Example

A developer checks whether every condition inside the login function executes correctly.

---

## Differences

| Black Box Testing | White Box Testing |
|-------------------|-------------------|
| Tests functionality | Tests internal code |
| No coding knowledge required | Coding knowledge required |
| Performed by QA | Performed by Developers |
| Focuses on input and output | Focuses on code paths and logic |
| User perspective | Developer perspective |

---

## Advantages of Black Box Testing

- Easy to perform.
- No programming knowledge required.
- Validates user requirements.

---

## Advantages of White Box Testing

- Finds logical errors.
- Improves code quality.
- Helps achieve better code coverage.

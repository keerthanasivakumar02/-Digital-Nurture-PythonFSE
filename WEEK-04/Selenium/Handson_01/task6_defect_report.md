# Task 6 - Defect Report

## What is a Defect Report?

A Defect Report (Bug Report) is a document used to record and communicate software defects found during testing.

A good defect report helps developers understand and fix the issue quickly.

---

## Defect Report Fields

- Defect ID
- Defect Title
- Module
- Environment
- Severity
- Priority
- Preconditions
- Steps to Reproduce
- Expected Result
- Actual Result
- Status
- Reported By
- Assigned To

---

## Sample Defect Report

### Defect ID

BUG_001

### Defect Title

Login button is not working with valid credentials.

### Module

Login Module

### Environment

- Windows 11
- Google Chrome
- Python Selenium

### Severity

High

### Priority

High

### Preconditions

Application is running.

### Steps to Reproduce

1. Open the Login page.
2. Enter a valid username.
3. Enter a valid password.
4. Click the Login button.

### Expected Result

The Dashboard should open successfully.

### Actual Result

The Login page remains the same and no action is performed.

### Status

New

### Reported By

QA Engineer

### Assigned To

Developer
